# Query Service v3 — True Agent with ReAct Pattern

## Tổng quan

v3 nâng cấp `query-service` từ **Guarded Tool Routing** (v2) thành **True Agent** sử dụng ReAct pattern. LLM điều khiển hoàn toàn việc quyết định có gọi tool hay không, gọi tool nào, với arguments gì. Backend chỉ enforce ACL/auth boundaries.

---

## Cách bật sử dụng

### Cách 1: Qua biến môi trường (.env)

Mở file `.env` trong `src/query-service/`, tìm hoặc thêm dòng:

```env
# Agent mode: guarded | react
# - guarded: Legacy Guarded Tool Routing (2 LLM calls, deterministic) — mặc định
# - react:   True Agent với ReAct pattern (dynamic 1-N LLM calls)
AGENT_MODE=react
AGENT_MAX_ITERATIONS=3
```

Restart lại service:

```bash
cd src/query-service
python -m uvicorn app.interfaces.api.main:app --reload
```

### Cách 2: Qua Docker / Docker Compose

```yaml
# docker-compose.yml
environment:
  - AGENT_MODE=react
  - AGENT_MAX_ITERATIONS=3
```

### Cách 3: Qua Kubernetes ConfigMap

```yaml
data:
  AGENT_MODE: "react"
  AGENT_MAX_ITERATIONS: "3"
```

## So sánh 2 chế độ

| | `AGENT_MODE=guarded` (mặc định) | `AGENT_MODE=react` |
|---|---|---|
| **Pattern** | Guarded Tool Routing | ReAct Agentic Loop |
| **Số LLM calls** | Cố định: 2 lần | Dynamic: 1-N lần |
| **Ai quyết định gọi tool** | Backend (hard-code logic) | LLM tự quyết định |
| **Loop / ReAct cycle** | Không có | Có — suy nghĩ rồi hành động rồi quan sát |
| **Replan khi tool fail** | Không | Có |
| **Reasoning trace** | Không | Có — structured logging |
| **Max iterations** | N/A | Có — mặc định 3, config được |
| **Latency** | Thấp (<500ms) | Cao hơn tùy query |
| **Độ phức tạp câu hỏi** | Đơn giản, 1 tool là đủ | Phức tạp, có thể cần nhiều tools |
| **Backward compatible** | — | Có — không break API cũ |

### Khi nào dùng chế độ nào?

| Tình huống | Nên dùng |
|---|---|
| Dev/test nhanh | `guarded` |
| Production ổn định, predictable | `guarded` |
| Câu hỏi phức tạp cần suy luận nhiều bước | `react` |
| Muốn LLM tự quyết định tool thay vì backend | `react` |
| Cần agentic reasoning (multi-step) | `react` |

## Cách hoạt động (REACT mode)

Khi bạn gửi 1 câu hỏi với `AGENT_MODE=react`:

```
Bạn: "Chính sách nghỉ phép năm nay thế nào? Và tôi còn bao nhiêu ngày?"

Agent suy nghĩ và hành động:
─────────────────────────────────────────────
Iteration 1:
  🤖 THOUGHT: "Câu hỏi có 2 phần — hỏi về chính sách (tài liệu)
                và số ngày nghỉ (HR data). Cần gọi 2 tools."

  🔧 ACTION: gọi rag_search
     → Backend inject document_ids từ ACL (user không thể truy cập doc lạ)
     → Kết quả: "Chính sách nghỉ phép: 12 ngày/năm,..."

  👁 OBSERVATION: "Đã có thông tin chính sách. Cần thêm HR data
                   cho số ngày nghỉ còn lại."

─────────────────────────────────────────────
Iteration 2:
  🤖 THOUGHT: "Đã có context về chính sách. Bây giờ cần gọi
               hr_query để lấy số ngày nghỉ còn lại."

  🔧 ACTION: gọi hr_query(intent="leave_balance")
     → Backend inject user_id từ auth token (LLM không bao giờ được set user_id)
     → Kết quả: "Còn 8 ngày nghỉ phép năm 2026"

  👁 OBSERVATION: "Đủ thông tin để trả lời cả 2 phần câu hỏi."

─────────────────────────────────────────────
Final:
  ✅ ANSWER: stream về người dùng qua SSE
```

## Kiểm tra Agent có đang chạy đúng mode

### Qua log

Khi có request, log sẽ cho biết mode:

```
# GUARDED mode:
agent_run_start agent.mode=GUARDED session_id=...

# REACT mode:
agent_run_start agent.mode=REACT session_id=...
agent_react_step iteration=0 is_final=False tool_calls_count=1
agent_react_step iteration=1 is_final=True
```

### Qua response event

```python
# Event cuối cùng (done=True) có thêm trường agent_mode:
{
    "done": True,
    "sources": [...],
    "session_id": "...",
    "outcome": "success",
    "agent_mode": "react",     # ← "guarded" hoặc "react"
    "iterations": 2            # ← chỉ có khi react mode
}
```

### Qua config

```bash
# Kiểm tra biến môi trường hiện tại
cd src/query-service
python -c "from app.infrastructure.config import get_settings; s = get_settings(); print(f'AGENT_MODE={s.agent_mode}, MAX_ITERATIONS={s.agent_max_iterations}')"
```

```env
# .env
LLM_MODE=openai
OPENAI_API_KEY=sk-proj-...
OPENAI_LLM_MODEL=gpt-4o-mini
AGENT_MODE=react
```

Nếu không có API key, agent sẽ dùng `MockAgentLLMClient` và fallback về mock answers.

## Troubleshooting

### Lỗi: "OPENAI_API_KEY is required for AGENT_MODE=react"

```bash
# Cài đặt API key
export OPENAI_API_KEY=sk-proj-...
# Hoặc đặt trong .env
OPENAI_API_KEY=sk-proj-...
```

### Lỗi: Agent gọi quá nhiều tools

```env
# Giảm max iterations
AGENT_MAX_ITERATIONS=2
```

### Lỗi: LLM không trả lời được

Kiểm tra log:
```bash
# Xem log agent trong terminal chạy uvicorn
# Tìm dòng: agent_react_step iteration=... is_final=...
```

### Không thấy khác biệt giữa guarded và react

Đảm bảo đã restart service sau khi đổi `AGENT_MODE`:

```bash
# Tắt service (Ctrl+C)
# Bật lại
python -m uvicorn app.interfaces.api.main:app --reload
```

## Architecture Summary

```
┌─────────────────────────────────────────────────────────┐
│                    User Query                             │
│              "Chính sách nghỉ phép?                       │
│               Và tôi còn bao nhiêu ngày?"                │
└────────────────────────┬────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────┐
│          QueryOrchestrationUseCase.stream()               │
│                                                         │
│   if AGENT_MODE == "react":                              │
│       → delegate to QueryAgentLoop.run()  ← REACT MODE   │
│   else:                                                 │
│       → legacy orchestration path       ← GUARDED MODE   │
└────────────────────────┬────────────────────────────────┘
                         │
          ┌──────────────┴──────────────┐
          │                              │
          ▼                              ▼
┌─────────────────────┐    ┌─────────────────────────────┐
│  REACT MODE         │    │  GUARDED MODE              │
│                     │    │                             │
│  LLM.react_step()   │    │  LLM: chọn tool_name      │
│    ↓                │    │  Backend: gọi tool        │
│  Backend ACL Guard  │    │  LLM: sinh answer         │
│    ↓                │    │                             │
│  Execute Tool       │    │  Luôn 2 LLM calls         │
│    ↓                │    │  Không có loop            │
│  LLM.react_step()   │    │                             │
│    ↓                │    │                             │
│  Stream Answer      │    │                             │
└─────────────────────┘    └─────────────────────────────┘
```
